# PR: Плейлисты — создание, отображение, добавление треков

## Обзор
Реализованы экран создания плейлиста, список плейлистов в Медиатеке и добавление трека в плейлист с экрана плеера (включая создание нового плейлиста «из плеера» с автоматическим добавлением текущего трека). Доработаны UI под светлую/тёмную темы и кастомный Toast по макету.

---

## Задача 1. Создать плейлист
- **Экран:** Fragment с полями обложка (photo picker), название*, описание, кнопка «Создать».
- Кнопка «Создать» неактивна при пустом названии; активное состояние — синяя кнопка по макету (drawable с закруглением 8dp).
- При нажатии на область обложки открывается выбор изображения (Photo Picker); при сохранении плейлиста файл копируется в приватное хранилище приложения (`filesDir/covers/`).
- Закрытие по «Назад»/системный Back: при наличии введённых данных показывается диалог «Завершить создание плейлиста?» с кнопками «Отмена» и «Завершить».
- **Данные:** Room — таблица `playlists` (id, name, description, coverPath, trackIdsJson, trackCount), таблица `playlist_tracks` для треков в плейлистах. DAO, репозиторий, интерактор; создание плейлиста возвращает id.
- Навигация: переход с Медиатеки (кнопка «Новый плейлист» в разделе «Плейлисты»); на экране создания нижняя панель навигации скрыта.

## Задача 2. Отобразить список плейлистов
- В разделе «Плейлисты» экрана «Медиатека» — список в две колонки (`GridLayoutManager`), элемент: обложка (или заглушка), название, количество треков.
- Пустое состояние: иконка и текст «Вы не создали ни одного плейлиста».
- После успешного создания пользователь возвращается на Медиатеку и видит новый плейлист в списке (Flow из БД).

## Задача 3. Добавить трек в плейлист
- На экране «Аудиоплеер» кнопка «Добавить в плейлист» открывает bottom sheet с заголовком «Добавить в плейлист», кнопкой «Новый плейлист» и списком плейлистов.
- Реализовано через `CoordinatorLayout`, overlay (#99000000) и `BottomSheetBehavior`; при открытии sheet затемняется контент плеера.
- По нажатию на плейлист: если трека нет в плейлисте — трек добавляется, sheet закрывается, Toast «Добавлено в плейлист [название]»; если трек уже есть — Toast «Трек уже добавлен в плейлист [название]».
- Кнопка «Новый плейлист» в sheet ведёт на экран создания плейлиста; **текущий трек передаётся аргументом и после создания автоматически добавляется в новый плейлист** (на пустом плейлисте отображается «1 трек»).

## Дополнительные доработки UI

### Экран создания плейлиста
- Область обложки без изображения: заглушка по макету (`ic_no_icon_playlist`).
- Кнопка «Создать»: по макету — серая (неактивная) и синяя (активная), закругление 8dp; отключён `backgroundTint`, чтобы в тёмной теме кнопка не становилась чёрной.
- **Тёмная тема:** цвет подсказок полей и обводки (default/focused) вынесены в ресурсы; рамка при фокусе — светлая подсветка (#E6E8EB), без синего.

### Плеер, bottom sheet
- Кнопка «Новый плейлист» в bottom sheet: в тёмной теме явно задан цвет текста `new_playlist_btn_text` (#1A1B22), чтобы текст был виден на белом фоне кнопки.

### Toast «Плейлист [название] создан»
- Кастомный Toast по макету (SVG): светлая тема — фон #1A1B22, белый текст; тёмная тема — белый фон, текст #1A1B22.
- Ширина на всю экран (`displayMetrics.widthPixels`), позиция по нижнему краю (y offset 0) — перекрывает нижнюю панель навигации.

## Технические изменения
- **Data:** `PlaylistEntity`, `PlaylistTrackEntity`, `PlaylistsDao`, `PlaylistTracksDao`, `PlaylistMapper`, `PlaylistTrackMapper`; `PlaylistsRepositoryImpl` (createPlaylist, getPlaylists, getPlaylistById, addTrackToPlaylist); копирование обложки в `filesDir/covers/`.
- **Domain:** модель `Playlist`, интерфейсы и реализации `PlaylistsRepository`, `PlaylistsInteractor` (в т.ч. `getPlaylistById`).
- **Presentation:** `CreatePlaylistFragment`/ViewModel, `PlaylistsFragment`/ViewModel, адаптеры для сетки плейлистов и для bottom sheet; обновлён `PlayerFragment`/ViewModel (bottom sheet, список плейлистов, добавление трека, передача трека при переходе в создание плейлиста).
- **Навигация:** `createPlaylistFragment` в графе с опциональным аргументом `trackToAdd`; действия с Медиатеки и с плеера; скрытие bottom nav на экране создания плейлиста.
- **Ресурсы:** строки, цвета (в т.ч. для тёмной темы), drawable кнопок и Toast, layout `toast_playlist_created.xml`.
